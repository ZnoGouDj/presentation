Garbage collection

Reachability

  • Local variables and parameters of the current function.
  • Variables and parameters for other functions on the current chain of nested calls.
  • Global variables.
  • (there are some other, internal ones as well)

Unreachable island

  • The garbage collector takes roots and "marks" them.
  • It visits and "marks" all references from them.
  • It visits marked objects and marks their references.
  • ...And so on until every reachable references are visited.
  • All objects except marked ones are removed.

Internal algorithms

Optimizations:

  • General collection
  • Incremental collection
  • Idle-time collection

Summary

The main things to know:

  • Garbage collection is performed automatically.
  • Objects are retained in memory while they are reachable.
  • Being referenced is not the same as being reachable (from a root).
